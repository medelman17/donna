## Key Decisions Made

### 1. Tool-result driven cards (not model-authored)

The model MUST NOT output UI specs. Cards are auto-generated server-side by intercepting `tool-result` events from `fullStream` and parsing the `gh_query` JSON response (raw GitHub API). This eliminates all inline JSON formatting discipline problems. The model writes pure markdown; the server produces structure from data.

### 2. Custom SSE over AI SDK data stream

`toTextStreamResponse()` / the default AI SDK data stream loses step boundaries — multi-step text gets concatenated without separators ("are.Solid" bug). `fullStream` exposes `step-start`, `tool-call`, `tool-result`, `text-delta` events, allowing us to insert `sep` events between steps and `tool-start`/`tool-end` indicators.

Custom SSE event protocol:
```
event: text\ndata: "..."\n\n
event: tool-start\ndata: {"name":"gh_query","input":{...}}\n\n
event: tool-end\ndata: {"name":"gh_query"}\n\n
event: card\ndata: {"type":"ProfileHeader","props":{...}}\n\n
event: sep\ndata: null\n\n
event: done\ndata: null\n\n
```

### 3. `gh_query` is the only structured tool

Other tools (`web_search`, `web_scrape`, `linkedin_lookup`) return plain strings. Only `gh_query` returns parseable GitHub API JSON, so `cardsFromToolResult()` only handles `gh_query`. Future: could add structured returns to other tools.

### 4. Partial text renders as plain text

During streaming, partial text is rendered without Markdown processing (avoids expensive re-parses on every token). Once a block is "sealed" (a separator or done event arrives), it switches to `<ReactMarkdown>`. This is intentional — acceptable UX tradeoff for smooth streaming.

## Failed Approaches & Gotchas

### Approach A: json-render standalone JSONL mode
Model outputs JSONL patches, server feeds to json-render. **Failed** because the model skips narrative ReasoningCards entirely — it goes straight to structured output and omits investigation commentary.

### Approach B: Inline mode (text + `{"card":"..."}` JSON lines)
Model writes text narrative and embeds card JSON on separate lines. **Failed** because: model doesn't maintain discipline (JSON leaks into text mid-sentence), missing spaces between steps, broken card parsing. Required an increasingly complex brace-depth parser (`01a7e8e`) that still couldn't fully contain the problem.

### `catalog.prompt()` — 16K system prompt
Too verbose. Includes state/actions/events the streaming flow doesn't use. Removed in favor of a stripped system prompt.

### `toTextStreamResponse()` for multi-step
Loses step boundaries. Text from step N and step N+1 gets concatenated without separator. This caused text like "are.Solid" where two sentences from different steps were joined. Fixed by switching to `fullStream` with manual SSE and inserting `sep` events in `onStepFinish`.

### React strict mode double-fetch
React strict mode mounts → unmounts → remounts in dev. Without cleanup, two concurrent `fetch` calls hit the enrichment API simultaneously. Fixed by storing `AbortController` in `useEffect` cleanup and aborting on unmount.
