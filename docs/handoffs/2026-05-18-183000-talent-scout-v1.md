## Key Decisions Made

1. **Dual agent architecture** — Python (Claude Agent SDK) for CLI batch enrichment, TypeScript (Vercel AI SDK) for web-triggered enrichment. Same tools, same DB, different runtimes. Reason: web needs native abort/streaming that Python subprocess can't provide.

2. **Custom SSE over AI SDK data stream** — `result.fullStream` → `for await...of` → custom `data: {...}` events. Reason: AI SDK v6's data stream protocol is opaque; custom SSE gives us typed events the client can parse reliably.

3. **pgvector column via raw SQL, not Prisma schema** — `Unsupported("vector(1536)")` breaks Prisma's shadow database during migrations. Column added via `ALTER TABLE` after migration.

4. **Redis pub/sub for Python → web streaming** — Python CLI publishes enrichment events to Redis channels; a separate SSE route (`/api/enrich/[login]/stream`) subscribes and pipes to the browser. This coexists with the TS agent's direct streaming.

5. **Firecrawl raw HTTP over SDK** — TypeScript tools use raw `fetch` to Firecrawl API instead of `firecrawl-js` SDK. Fewer deps, more control.

## Failed Approaches & Gotchas

See `docs/learnings/2026-05-18-full-stack-agent-platform.md` for 9 detailed gotchas including:
- Firecrawl SDK `.web` vs `.data` (broke web search)
- Prisma pgvector shadow DB issue
- psycopg transaction rollback requirement
- AI SDK v6 API surface changes (3 separate fix commits)
- React 19 strict mode double-invoke aborting fetches
